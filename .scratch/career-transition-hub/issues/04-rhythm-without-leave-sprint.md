# Rhythm without leave sprint

Type: grilling
Status: resolved
Blocked by:

## Question

What is the **daily/weekly rhythm model** for phase B (and how it changes in phase A) once the 29-day leave frame is gone?

Decide cadence objects (day plan vs rolling habits vs weekly themes), which disciplines are first-class, and how rhythm ties to readiness evidence without becoming another abandoned checklist. Leave-sprint’s four-block rhythm is reference only — not sacred.

## Answer

**Cadence:** rolling **daily checklist** + **weekly review** (no 29-day spine, no leave-era day plans as product truth).

**Phase B first-class disciplines (daily slots):**
1. **Practice** — coding/patterns  
2. **Defense** — stories / file defense  
3. **Interview reps** — Q / mock / rubric work  
4. **Admin light** — plan, log, glance at gate — not “build the product” as a rhythm cell  

**Phase A:** same four slots; **mode-weighted** — Admin light expands (apps touch expected most days); Practice/Defense/Interview may shrink without failing the day; weekly review prioritizes pipeline + interviews. No separate second rhythm product; optional fifth “Applications” slot rejected.

**Evidence link:** rhythm = **did the work**; evidence floor = **is it good enough**. Checkboxes never auto-green the hybrid gate. Weekly review can surface distance-to-floor. Logging concrete artifacts (e.g. a mock) may feed evidence records; a bare “Interview reps” check is not itself a solid mock.
