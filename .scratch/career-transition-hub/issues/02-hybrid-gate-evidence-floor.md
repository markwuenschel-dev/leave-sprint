# Hybrid gate evidence floor

Type: grilling
Status: resolved
Blocked by:

## Question

What is the **checkable evidence floor** for phase B → phase A, given the hybrid gate (evidence minimum **and** personal go/no-go)?

Lock the dimensions and rough thresholds worth tracking (e.g. pattern coverage, mock quality, rubric coverage on SWE FS II and MLE II, file-defense cold recall) — not every metric UI — so readiness dashboards and next-action ranking have a spine. Explicitly list what is **not** required for the floor.

## Answer

**Gate type:** hybrid — hub can show **evidence green** only when the floor is met; **you** still flip phase B → A (personal go/no-go). No auto-applications.

**Floor dimensions** (all three required):
1. **Practice solidity** — core pattern/problem list mostly **Solid** (~80% of a defined core list, not “every problem ever”)
2. **Interview performance** — **≥2** solid mocks or equivalent scored sessions at/above a bar (exact bar is later detail)
3. **Story / file defense cold** — a **short fixed core list** deliverable without notes

**Scope:** floors apply **per primary role** — **SWE Full Stack II** and **MLE II** both must clear all three. Secondary (DS, DE) and escape (BIE/BIA) **never block** evidence green.

**Not on the floor (explicit):** application counts, network size, resume/materials polish, perfect secondary/escape roles, full qbank completion, “feel ready” as a tracked metric (that’s the personal go/no-go only).

**Deferred to later tickets/implementation:** exact core lists, mock score bar numbers, UI for the gate.
