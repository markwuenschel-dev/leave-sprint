# Hub information architecture

Type: prototype
Status: resolved
Blocked by: 01, 02, 03, 04

## Question

What is the **top-level IA / navigation shell** for the hub so phase B and phase A both fit one product without a leave-sprint tab dump?

Produce a cheap prototype asset (outline, wire IA, or rough shell) covering primary nav, where readiness gate lives, where applications live, and how dual-primary roles appear. React to it with the human; revise once if needed. Depends on name, evidence floor, applications surface, and rhythm model so the shell isn’t speculative fiction.

## Answer

**Stamped defaults** (human authorized agent recs; no live prototype review cycle).

Primary nav: **Today · Readiness · Practice · Defense · Interview · Applications · Weekly · More**.

- Gate lives on **Readiness** (+ chip on Today).
- Apps on **Applications** (wishlist allowed in phase B).
- Dual primaries: side-by-side on Readiness; role filter on practice surfaces.
- Same shell for B and A; mode-weighted Today/Weekly — not two apps.
